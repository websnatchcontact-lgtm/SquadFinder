1. REQUEST TO JOIN VALIDATION (STRICT BUSINESS RULE)
==============================================================================

Currently there is a bug.

When a student submits a "Request to Join", the application allows requests even when the student's specialization does not match the specialization of the target group.

This should NEVER happen.

This validation must be enforced before the request is created.

Example

Target Group
Specialization:
AI/ML

Student Request

Specialization:
CS

Result

❌ Request must NOT be submitted.

Instead display a professional validation message.

Example

"You cannot request to join this group because it belongs to the AI/ML specialization while your specialization is CS.

Capstone groups can only contain students from the same specialization."

The request should NOT be stored.

The request should NOT appear in Pending Requests.

The request should NOT update Local Storage.

The request should NOT update statistics.

The modal should remain open so the user can correct the specialization.

------------------------------------------------------------------------------

Valid Examples

AI/ML → AI/ML

✓ Allow

CS → CS

✓ Allow

------------------------------------------------------------------------------

Invalid Examples

CS → AI/ML

❌ Block

AI/ML → CS

❌ Block

This validation should happen BEFORE any request is saved.

Users should also be able to revoke (withdraw) their own pending request.

Add a new action.

Button

"Revoke Request"

Place it beside the existing Pending badge inside every Pending Request card.

Example

------------------------------------

Pending

[ Revoke Request ]

------------------------------------

Title

Withdraw Request?

Message

"Are you sure you want to revoke this join request?

Buttons

Cancel

Revoke Request

------------------------------------------------------------------------------

After confirmation

Remove the request immediately.

Update Local Storage.

Update the Dashboard.

Update Group Details.

Update Pending Request count.

Update statistics.

Remove the request from every place where it appears.

Show success toast.
