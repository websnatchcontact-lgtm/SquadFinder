==============================================================================
DATA ARCHITECTURE
==============================================================================

The application should maintain two separate sources of data.

--------------------------------------------------

Source 1

Original Demo Data

students.json

Read Only

Never modify.

--------------------------------------------------

Source 2

Local Storage

Contains

• Created Groups

• Pending Join Requests

• Group Confirmations

• Group Notes

• Conflict Records

• Available Students

• User Preferences (if any)

--------------------------------------------------

Application Layer

Always merge

Original JSON

+

Local Storage

into one unified application state.

Never overwrite the original dataset.

Never duplicate data unnecessarily.

==============================================================================

EDGE CASES
==============================================================================

Handle all edge cases gracefully.

Example

Searching for a student that doesn't exist.

Display a friendly empty state.

--------------------------------------------------

Attempting to create a group with only one member.

Display validation.

--------------------------------------------------

Attempting to exceed four members.

Prevent the action.

--------------------------------------------------

Attempting to add the same enrollment number twice inside one group.

Prevent the action.

--------------------------------------------------

Attempting to mix CS and AI/ML students.

Prevent the action.

--------------------------------------------------

Attempting to request to join a full group.

Disable the Request To Join button.

Explain why.

--------------------------------------------------

Local Storage corrupted.

Automatically recover.

Keep original demo data safe.
