# IMPORTANT

Everything in this document extends the existing application.

Do NOT redesign existing pages.

Do NOT replace components.

Integrate the following features into the existing architecture.

The UI should feel as though these features always existed.

==============================================================================

# APPLICATION PHILOSOPHY

The application should NEVER decide who is right.

The application should NEVER decide which group is correct.

The application should NEVER automatically remove students.

Instead,

the application should simply collect information,

detect conflicts,

and clearly visualize them.

The final decision always belongs to the students.

==============================================================================

# GROUP CONFIRMATION SYSTEM

Introduce a confirmation system.

Purpose:

Allow students to confirm that they actually belong to a group.

This increases confidence in group accuracy.

--------------------------------------------------

Creator

When a group is created,

the creator is automatically marked as:

Confirmed

Reason:

The creator intentionally created the group.

--------------------------------------------------

Other Members

Every other member starts as:

Unconfirmed

They remain unconfirmed until they manually confirm.

==============================================================================

# CONFIRMATION FLOW

When any student searches themselves,

and they appear inside a group,

display a prominent action.

Button:

"I Confirm I'm In This Group"

Clicking this should:

Mark the student as Confirmed.

Update dashboard statistics immediately.

Update group progress immediately.

Persist inside Local Storage.

Show success toast.

Example

✓ You are now confirmed in Group 8.

==============================================================================
