# PRIMARY OBJECTIVE

The original SquadFinder application only allows students to search and browse groups.

The application must now evolve into a collaborative Capstone Group Coordination Platform.

The application must still remain:

• Frontend only

• No Backend

• No Authentication

• No Database

• No APIs

Everything should continue working entirely using:

• Static JSON

+

• Browser Local Storage

# CORE BUSINESS MODEL

This application is NOT an official college management system.

It is a student collaboration platform.

The application does NOT decide which group is correct.

The application only records and visualizes information entered by students.

Whenever conflicting information exists,

the application should clearly display the conflict,

but should NEVER attempt to automatically resolve it.

Display a modern dialog explaining:

"This group contains students from different specializations.

# DIVISION RULE

The college contains two divisions.

Division A

Division B

Unlike specialization,

Division has NO restriction.

The following combinations are valid:

A + A

A + B

B + A

B + B

Division is informational only.

Division should appear on:

Student Card

Search Result

Group Details

Dashboard

Conflict Details

Statistics

# ENROLLMENT NUMBER

Enrollment Number is the PRIMARY IDENTIFIER.

This is extremely important.

Never identify students using Name.

Never identify students using Index.

Never identify students using Array Position.

Only Enrollment Number uniquely identifies a student.

Enrollment Number validation:

✓ Numeric only

✓ Remove whitespace automatically

✓ Reject alphabets

✓ Reject symbols

✓ Reject emojis

✓ Reject punctuation

No fixed length validation.

Apply this validation everywhere:

Search

Create Group

Add Member

Join Request

Conflict Detection

Confirmation

Future Features

# CROSS GROUP DUPLICATES

A student MAY appear in multiple different groups.

This is allowed.

However,

doing so automatically creates a conflict.

Whenever this happens,

show a professional confirmation dialog.

"This student already belongs to another registered group.

Continuing will create a conflict that will be visible on the dashboard."

Options

Cancel

Continue Anyway

Never silently create conflicts.
